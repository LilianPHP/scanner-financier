M3 — Intelligence
Planifié

Passer de règles mots-clés statiques (~70% précision) à un modèle ML entraîné (~90%+). Applicable quelle que soit la source des données — CSV ou API bancaire.

Dataset d'entraînementPlanifié
Transactions en base Supabase + règles existantes comme oracle + datasets Kaggle. Cible : 20k–50k exemples étiquetés.
1 semaine Prérequis ML
Modèle TF-IDF + XGBoostPlanifié
N-grammes de caractères sur libellés bruts. Robuste aux abréviations bancaires. Modèle ~15 MB, inference <1ms, déployable sur Railway sans surcoût.
1–2 semaines ~90% précision
Score de confiancePlanifié
Si confidence < 0.6 → fallback Claude API. Réduction massive des "autres". Amélioration continue via corrections users.
2–3 jours IA fallback
Apprentissage continuPlanifié
